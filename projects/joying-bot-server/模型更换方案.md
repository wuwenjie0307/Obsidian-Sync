## VoxCPM 部署记录 ✅

### 部署环境

| 项目 | 值 |
|------|-----|
| 服务器 | h20（223.112.222.90）/ hgx19 |
| Python | 3.13.5（系统 anaconda3，未用 conda 隔离） |
| CUDA | 12.4 |
| GPU | NVIDIA H20 96GB（第0块卡） |
| 版本 | voxcpm 2.0.3（VoxCPM2 模型） |
| 模型路径 | `/root/.cache/huggingface/hub/models--openbmb--VoxCPM2/` |
| 模型大小 | ~5 GB |
| 采样率 | 48kHz |

### 安装步骤

```bash
pip install voxcpm
# 首次运行自动下载 VoxCPM2 模型权重（~5GB）
```

### 踩坑记录

| 问题                              | 原因                                                | 解决                                                                               |
| ------------------------------- | ------------------------------------------------- | -------------------------------------------------------------------------------- |
| GPU 不可用，回退到 CPU                 | pip 自动装的 PyTorch 2.12 需要 CUDA 12.8，但服务器驱动只支持 12.4 | 降级：`pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cu124` |
| torchaudio 报 libcudart.so.13 缺失 | torchaudio 2.11 与 torch 2.5 不兼容                   | 重装 torchaudio（pip 自动匹配版本）                                                        |
| 生成音频 4 秒                        | 正常，文本短对应音频短                                       | —                                                                                |

### CLI 使用

```bash
# 先连接 h20（从生产服务器跳板：sudo su → ssh root@h20 -p 10019）

# 基础合成
voxcpm --text "要合成的文本" --output /tmp/output.wav

# 语音克隆（需要参考音频）
voxcpm --text "要合成的内容" \
  --prompt-audio /path/to/voice.wav \
  --prompt-text "参考音频对应的文本" \
  --output /tmp/output.wav
```

### Python API

```python
from voxcpm import VoxCPM
model = VoxCPM.from_pretrained("openbmb/VoxCPM2")
wav = model.generate(text="你好世界", prompt_wav_path="speaker.wav", prompt_text=None)
```

### 部署环境

| 项目 | 值 |
|------|-----|
| 服务器 | h20（223.112.222.90）/ hgx19 |

| 版本 | LatentSync 1.6 |
| 代码路径 | `/root/LatentSync/` |
| 模型路径 | `/root/LatentSync/checkpoints/` |
| 模型大小 | latentsync_unet.pt 4.8 GB + whisper/tiny.pt ~72 MB |
| 推理显存 | 18 GB（1.6版） |

### 安装步骤

```bash
# 1. 创建 conda 环境
conda create -n latentsync python=3.10.13 -y
conda activate latentsync

# 2. 安装 ffmpeg
conda install -y -c conda-forge ffmpeg

# 3. 安装系统依赖
sudo apt -y install libgl1

# 4. 安装 Python 依赖
cd /root/LatentSync
pip install -r requirements.txt

# 5. 下载模型权重（国内用镜像加速）
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download ByteDance/LatentSync-1.6 whisper/tiny.pt --local-dir checkpoints
huggingface-cli download ByteDance/LatentSync-1.6 latentsync_unet.pt --local-dir checkpoints
```

### 关键依赖版本

| 包 | 版本 |
|------|------|
| torch | 2.5.1+cu121 |
| torchvision | 0.20.1+cu121 |
| diffusers | 0.32.2 |
| transformers | 4.48.0 |
| insightface | 0.7.3 |
| onnxruntime-gpu | 1.21.0 |
| gradio | 5.24.0 |

### 启动方式

```bash
# 1. 连接 h20（从生产服务器跳板）
ssh developer@222.71.55.27 -p 9527   # 先连生产服务器
sudo su                                # 切 root
ssh root@h20 -p 10019                  # 跳板到 h20

# 2. 激活环境
conda activate latentsync
cd /root/LatentSync
export HF_ENDPOINT=https://hf-mirror.com   # 国内加速

# 3. 命令行推理
python -m scripts.inference \
    --unet_config_path "configs/unet/stage2_512.yaml" \
    --inference_ckpt_path "checkpoints/latentsync_unet.pt" \
    --inference_steps 20 \
    --guidance_scale 1.5 \
    --enable_deepcache \
    --video_path "input.mp4" \
    --audio_path "input.wav" \
    --video_out_path "output.mp4"

# 4. Gradio Web 界面（调试用）
python gradio_app.py
```

### 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| HuggingFace 下载极慢 | 国内直连慢 | `export HF_ENDPOINT=https://hf-mirror.com` 用镜像加速 |
| ONNX Runtime CUDA 加载失败 | 缺少 `libnvrtc.so.12` | InsightFace 人脸检测回退 CPU，不影响结果 |

### 测试结果

- Demo 视频推理成功，每步约 5.8 秒，16 步共 ~93 秒
- 输出：`video_out.mp4`
