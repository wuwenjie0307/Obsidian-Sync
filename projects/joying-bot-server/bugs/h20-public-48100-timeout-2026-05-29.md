# h20 公网 48100/48101 访问超时

## 问题描述

h20 Bot 已在服务器内部启动并监听 `0.0.0.0:48100`，CRM 路由本机验证通过，但从外部访问 `223.112.222.90:48100` 和 `223.112.222.90:48101` 仍然 TCP 超时。

## 原因

当前证据指向上游公网 NAT/ACL/端口映射未生效或未映射到预期 h20 内部端口：

- h20 本机没有公网 IP `223.112.222.90`，只有内网地址，例如 `172.16.220.119`。
- h20 Bot 本机 `127.0.0.1:48100` 和 `172.16.220.119:48100` 可访问。
- h20 本机防火墙不是阻塞点：`firewalld` inactive、`ufw` inactive、INPUT policy accept。
- 从本地外部网络访问 `223.112.222.90:48100`、`223.112.222.90:48101` 返回 `HTTP:000`，curl 超时。

## 复现步骤

1. h20 上确认 Bot 监听：`ss -ltnp | grep :48100`。
2. h20 本机验证：`curl http://127.0.0.1:48100/status/check` 返回 200。
3. 外部机器验证：`curl --connect-timeout 10 http://223.112.222.90:48100/status/check`。
4. 外部请求超时，无 HTTP 响应。

## 期望行为

公网 `223.112.222.90:48100` 能转发到 h20 内部 `172.16.220.119:48100`，CRM 测试服可访问 Bot `/crm/*` 接口。

## 实际行为

公网访问 `223.112.222.90:48100/48101` 超时，流量没有到达 h20 Bot。

## 环境信息

- h20 主机名: `hgx19`
- h20 内网 IP: `172.16.220.119`
- Bot 内部端口: `48100`
- VoxCPM 内部端口: `8100`
- LatentSync 内部端口: `8101`
- 公网 IP: `223.112.222.90`

## 修复方案

待确认。需要晋良或运营商确认以下信息：

- `223.112.222.90:48100` 是否已经映射到 h20 内部 `172.16.220.119:48100`。

- 是否存在仅允许特定来源访问的 ACL。
