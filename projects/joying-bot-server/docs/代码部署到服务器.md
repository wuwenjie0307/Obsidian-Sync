# 代码部署到服务器

## SCP 上传文件

从本地 Windows 上传文件/文件夹到生产服务器（222.71.55.27）：

```bash
# 上传文件夹（-r 递归，-P 指定端口）
scp -P 9527 -r C:\Users\admin\Desktop\<文件夹名> developer@222.71.55.27:/tmp/

# 上传单个文件
scp -P 9527 C:\Users\admin\Desktop\<文件名> developer@222.71.55.27:/tmp/
```

上传后在服务器上移动到目标目录：

```bash
sudo mv /tmp/<文件夹名> /data/project/
sudo chown -R joying:joying /data/project/<文件夹名>
```

## 目录权限

- `/data/project/` 属于 `joying:joying`，developer 用户无写权限
- 新建项目目录需 `sudo`，然后 chown 给 joying

## 测试 vs 生产

| 环境 | 目录 | supervisor 进程 |
|------|------|-----------------|
| 测试 | `/data/project/test_ai_autodone/` | `ai_autodone_py_test` |
| 生产 | `/data/project/prod_ai_autodone/` | `ai_autodone_py_prod` |

配置可能有差异，部署前先确认 `config.json` 和环境变量。

## SCP 上传到 h20（跳板中转）

h20 无法直连，需通过跳板机中转文件：

```bash
# 步骤1：本地 Windows → 222.71.55.27
scp -P 9527 -r C:\Users\admin\Desktop\<文件夹名> developer@222.71.55.27:/tmp/

# 步骤2：在 VS Code SSH 终端（222.71.55.27）上传到 h20
sudo scp -P 10019 -r /tmp/<文件夹名> root@h20:/data/projects/
```

### 部署步骤

```bash
# 1. 本地 clone test 分支
git clone -b test https://git.joyingai.cn/services/crm.ai.joyingbot.git

# 2. SCP 上传到 h20（两跳）
scp -P 9527 -r joyingbot-new developer@222.71.55.27:/tmp/
# 在 222.71.55.27 终端:
sudo scp -P 10019 -r /tmp/joyingbot-new root@h20:/data/projects/

# 3. 在 h20 上创建 conda 环境
/data/server/anaconda3/bin/conda create -n joyingbot python=3.10.13 -y
conda activate joyingbot

# 4. 修复 requirements.txt（移除 Windows 包）
grep -v "pywin32\|pypiwin32" requirements.txt > requirements.tmp && mv requirements.tmp requirements.txt

# 5. 安装依赖
pip install -r requirements.txt

# 6. 降级 torch/torchaudio（适配 CUDA 12.4）
pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cu124
pip install torchaudio==2.5.1 --index-url https://download.pytorch.org/whl/cu124

# 7. 启动 API 服务
conda activate joyingbot
cd /data/projects/joyingbot-new
python api_server/main.py --port 8100
```

### 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| h20 DNS 不解析 git.joyingai.cn | 公司内网域名，h20 不在内网 | 本地 clone 后 SCP 两跳上传 |
| pywin32/pypiwin32 Linux 安装失败 | Windows 专属包 | 从 requirements.txt 移除 |
| torch 2.12.0 CUDA 不可用 | voxcpm 装的最新 torch 需 CUDA 12.8，服务器驱动 12.4 | `pip install torch==2.5.1+cu124` |
| torchaudio libcudart.so.13 缺失 | torchaudio 2.11 不兼容 torch 2.5 | `pip install torchaudio==2.5.1+cu124` |
| git ambiguous 'test' 分支/目录冲突 | 项目有 test/ 目录名与分支冲突 | 用 `origin/test` 代替 `test` |
| conda 命令找不到 | `/bin/sh` PATH 不含 `/data/server/anaconda3/bin/` | 用完整路径 |
