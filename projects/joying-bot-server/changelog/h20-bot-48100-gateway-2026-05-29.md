# h20 Bot 复用 48100 公网入口

## 改动类型

- [ ] 新功能
- [ ] Bug 修复
- [ ] 重构
- [x] 配置变更

## 最新结论

当前目标不是让生产服调用 h20 模型，而是让 h20 测试服完整跑通 Bot + VoxCPM + LatentSync，并让外部 CRM 测试服能访问 h20 Bot 的 `/crm/*` 接口。

在不新增公网端口映射的约束下，采用以下方案：

| 用途 | h20 内部地址 | 外部地址 |
|---|---|---|
| Bot 主服务 `/crm/*` | `http://127.0.0.1:8100` | `http://223.112.222.90:48100` |
| VoxCPM API | `http://127.0.0.1:8110` | 不作为外部联调入口 |
| LatentSync API | `http://127.0.0.1:8101` | `http://223.112.222.90:48101` 可保留 |

## 配置目标

h20 Bot 的 `config/config-dev.json` 只补丁式修改以下字段，不能整文件覆盖远程配置：

```json
{
  "server_port": 8100,
  "h20_api_base": "http://127.0.0.1:8110",
  "voxcpm_api_base": "http://127.0.0.1:8110",
  "latentsync_api_base": "http://127.0.0.1:8101"
}
```

## 验证入口

- h20 本机 Bot 健康检查：`curl -s http://127.0.0.1:8100/status/check`
- 外部 Bot 健康检查：`curl -s http://223.112.222.90:48100/status/check`
- 外部试听接口：`POST http://223.112.222.90:48100/crm/voice_clone_audition`

试听接口要求 `voice_file_url` 是 h20 能访问到的公网或内网可达音频 URL。

## 影响范围

- `48100` 从 VoxCPM 外部入口改为 Bot 外部入口。
- h20 内部 Bot 仍通过 localhost 调模型，不依赖生产服访问 h20 模型。
- 生产服模型方案不受影响。
